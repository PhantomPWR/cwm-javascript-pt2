Object Oriented Programming

Objects

  1. If a function is part of an object, it's called a method
  2. If an object has one or more methods, it has behavior
  3. Objects can have properties and methods

Factory Functions

  1. Factory functions are functions that return objects
  2. Factory functions are camelCase
  3. Factory functions are used to create multiple objects
  4. Factory functions are used to create objects with different values

Constructor Functions

  1. Constructor functions are used to create objects
  2. Constructor functions are PascalCase
  3. Constructor functions are used to create a single object
  4. Constructor functions are used to create objects with the same values
  5. Constructor functions are used to create objects with behavior
  6. No need to return an object in a constructor function
  7. If 'new' is omitted, 'this' will refer to the global/window object

Value vs Reference Types

  1. Primitive types are copied by their value
  2. Reference types are copied by their reference
  3. Objects are reference types
  4. Objects are stored in memory
  5. Objects are copied by their reference
  6. Objects are mutable

Abstraction

  1. Abstraction is hiding the details and showing the essentials
  2. Used to reduce complexity
  3. Used to manage complexity
  4. Reduces impact of change

Private Properties & Methods

  1. Not accessible from outside
  2. Used to hide information
  3. Used to protect information
  4. Used to prevent bugs

Scope & Closure

  1. When you have a function inside another function, the inner function has access to the variables in the outer function. This is called closure.
  2. Scope is about where variables can be accessed
  3. Closures are about a function's ability to retain access to variables from its scope at creation time, even if that scope is no longer active
